# TaskFlow Product Spec

## 产品说明
- 一句话描述：面向自由职业者的轻量任务管理工具，支持 AI 自动拆解任务
- 核心价值：让自由职业者不再手动拆解复杂项目，AI 根据项目描述自动生成可执行的任务清单

## 功能清单

### 核心功能
- [ ] 项目创建：用户输入项目名称和一段自然语言描述，系统创建项目
- [ ] AI 任务拆解：用户点击"拆解"，AI 根据项目描述生成 3-8 个子任务，每个子任务包含标题、预估时间、优先级
- [ ] 任务状态管理：每个任务支持"待办 → 进行中 → 已完成"三种状态切换
- [ ] 任务手动编辑：用户可以修改 AI 生成的任务标题、预估时间、优先级，也可以手动添加/删除任务

### 辅助功能
- [ ] 项目列表：首页展示所有项目，按最近活跃排序
- [ ] 任务筛选：按状态（待办/进行中/已完成）筛选任务
- [ ] 简单统计：项目详情页顶部显示完成率（已完成/总数）

### 不做（MVP 边界）
- 多人协作和权限管理
- 日历视图和甘特图
- 任务之间的依赖关系
- 移动端适配（第一版只做桌面端 Web）
- 第三方集成（Slack、Notion 等）

## 用户流程

```
首页（项目列表）
  ├─ 点击"新建项目" → 输入名称和描述 → 创建成功 → 进入项目详情
  └─ 点击已有项目 → 进入项目详情
        ├─ 点击"AI 拆解" → 显示 loading → AI 返回子任务列表 → 用户确认/编辑
        ├─ 点击任务 → 修改标题/时间/优先级
        ├─ 拖拽或点击切换任务状态
        └─ 点击"添加任务" → 手动创建新任务
```

**异常流程：**
- AI 拆解失败（网络错误/API 超时）→ 显示错误提示 + 重试按钮
- 项目描述为空时点击"AI 拆解" → 按钮置灰，tooltip 提示"请先填写项目描述"

## 数据模型

### 核心实体
- **Project**: id, name, description, created_at, updated_at
- **Task**: id, project_id, title, status(todo/in_progress/done), priority(high/medium/low), estimated_minutes, created_by(ai/manual), created_at, updated_at

### 关键关系
- Project : Task = 1 : N
- 删除 Project 时级联删除所有 Task

## 第三方依赖
- AI 接口：Claude API（任务拆解）
- 无其他外部服务依赖

## 非功能需求
- AI 拆解响应时间 < 5 秒
- 支持 100 个项目 / 每项目 50 个任务的数据量（localStorage 或轻量数据库均可）
- 不需要用户登录（单用户本地使用）

## AI 系统提示词

### 角色定义
你是一个任务拆解助手。用户会给你一段项目描述，你需要将其拆解为可执行的子任务。

### 工作流程规则
1. 读取用户提供的项目描述
2. 生成 3-8 个子任务，每个包含：标题（动词开头，< 20 字）、预估时间（分钟）、优先级（high/medium/low）
3. 按优先级从高到低排列

### 回复规范
- 只返回 JSON 数组，不要解释
- 格式：[{"title": "...", "estimated_minutes": 30, "priority": "high"}, ...]

### 约束条件
- 最少 3 个任务，最多 8 个
- 预估时间最小 15 分钟，最大 480 分钟
- 如果项目描述过于模糊（< 10 个字），返回 {"error": "描述太简短，请补充更多细节"}
